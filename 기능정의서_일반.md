# 기능정의서 (일반 버전)
**유선 브로드밴드 품질 대시보드** · 작성일 2026-06-18 · 기준: 현재 구현 상태

---

## 1. 개요
- **목적:** 글로벌 오픈 API(Cloudflare Radar, M-Lab, RIPE Atlas) 기반 데이터로 국내·해외 유선 브로드밴드 ISP의 품질을 비교·추적하는 임원 보고용 대시보드.
- **대상:** NW 운용 / NW 관리 부서.
- **스택:** React + Vite + TypeScript, ApexCharts.
- **데이터 방식:** 프론트엔드는 정제된 단일 정적 파일 `quality_data.json` 만 페치한다. 라이브 API를 브라우저에서 직접 호출하지 않는다(토큰 노출·CORS·요청 한도 회피).
- **현재 데이터:** 시뮬레이션(모크) 데이터. 실 API 연동은 데이터 생성기 교체로 전환.

## 2. 화면 구성
- **상단 툴바:** 출처 선택, 기간 선택, 집계 단위 선택, API 설정, 스크린샷, 테마 전환.
- **좌측 패널:** 통신사 멀티셀렉트 드롭다운 + 데이터 생성 정보.
- **본문:** 선택한 출처에 속한 모든 지표를 세로로 스택(차트 + 피크타임 분석), 스크롤로 열람.

## 3. 기능 목록

### F-01 데이터 출처 전환
- 드롭다운으로 **Cloudflare Radar / M-Lab (ndt7·BigQuery) / RIPE Atlas** 3개 출처를 전환.
- 출처를 바꾸면 본문 지표 세트 전체가 교체된다.

### F-02 출처별 전체 지표 일괄 표시 (스크롤)
- 선택 출처의 모든 지표를 한 페이지에 세로로 나열. 지표를 개별 선택할 필요 없음.
- 출처별 지표:
  - **Cloudflare:** 지연시간(RTT), 지터, 대역폭(기준), HTTP 오류율
  - **M-Lab:** 평균 처리량, 최소 RTT, 손실률, 혼잡 윈도우, 페이싱 레이트
  - **RIPE Atlas:** Ping 평균 RTT, 가용성, 트레이스라우트 홉 수, AS 경로 변동, DNS 응답시간

### F-03 통신사 멀티셀렉트 (드롭다운)
- 단일 접이식 패널, 최대 80vh 내부 스크롤.
- **한국 통신사 상단 고정**(LG U+, KT, SK 브로드밴드), 해외는 국가별 그룹(접기 가능).
- 이름/ASN 검색, "표시 항목 선택 / 전체 해제".
- SK 브로드밴드의 선택적 `AS9644`(SKT 모바일) 포함 토글(유선 데이터 오염 방지, 기본 비활성).

### F-04 기간(범위) 선택
- 최근 24시간 / 2일 / 7일 / 30일 / 90일 / 180일 / **1년**.
- 단일 파일 + 1년 범위를 위해 다중 해상도(티어)를 자동 선택: 10분(2일)·1시간(30일)·1일(365일).

### F-05 집계 단위(버킷) 선택
- 10분 / 1시간 / 1일. 선택한 기간(티어)의 기본 해상도 이상만 노출.

### F-06 데이터 신뢰도 처리 (정제)
- **절단평균:** 각 버킷 표본을 정렬해 상·하위 5%를 제거하고 평균·중앙값 계산.
- **하드 임계값 필터:** 물리적으로 불가능한 값(예: 지연 500ms 초과) 제거.
- **저표본 검증:** 표본이 임계값(10분 10개, 1일 50개) 미만이면 경고 마커로 표시.

### F-07 피크타임 성능 저하 분석
- 혼잡(21–23시) vs 한산(02–05시) 대비.
- 통신사별 **처리량 방어율(%)** / **지연 급증률(%)** + 혼잡·한산 평균.
- 시간대 비교를 위해 항상 최근 고해상도(10분) 데이터를 사용.

### F-08 데이터 추적성 툴팁
- 차트 좌표 hover 시: 값, 중앙값, **총 샘플 / 절단 샘플 / 순 잔존율(%)**, 저표본 경고 표시.

### F-09 다크/라이트 테마
- 툴바 토글로 즉시 전환, 차트도 함께 변환. 선택 상태는 브라우저에 저장(재방문 유지).

### F-10 API / 데이터 소스 설정
- "데이터 소스 URL"(실제 페치 대상) + 제공자별 토큰(Cloudflare/M-Lab/RIPE) 입력·사용 토글.
- 브라우저에 저장. 운영에서는 토큰을 서버리스 수집기에서만 사용해야 함(보안 안내 표기).

### F-11 화면 스크린샷
- 현재 보고 있는 대시보드 화면을 PNG로 캡처·다운로드(파일명에 타임스탬프).

## 4. 데이터 사양
- **파일:** 단일 `public/quality_data.json` (다중 해상도, 약 10MB).
- **구조:** 출처 무관 통합. ISP × 지표 × 티어(fine/mid/coarse) 컬럼형 배열 `[값, 총샘플, 잔존샘플]`.
- **갱신(설계):** GitHub Actions 크론(10분)이 파일을 재생성·커밋(FR-07). 현재는 모크 생성기로 생성.

## 5. 비기능 요구
- **보안:** 자격증명은 클라이언트 번들에 미포함(서버리스 환경변수 전용 설계).
- **성능/비용:** 정적 파일 페치로 API 요청 한도(예: M-Lab BigQuery 1TB) 및 429 회피.
- **테마 적응성:** 전역 테마 전환 시 차트가 새로고침 없이 동시 변환.

## 6. 실행 / 배포
- **실행:** 바탕화면 「브로드밴드 대시보드」 바로가기 또는 `대시보드 실행.bat` 더블클릭(구성요소·데이터 자동 준비 후 브라우저 자동 오픈).
- **개발:** `npm install` → `npm run mock` → `npm run dev`. 빌드: `npm run build`.
- **전제:** PC에 Node.js(LTS) 설치 1회.

## 7. 제약 / 한계
- 현재 데이터는 시뮬레이션이며, 실 API 연동은 데이터 생성기(`scripts/generate-mock.ts`)의 생성 로직 교체가 필요.
- 저표본 "점선 세그먼트"는 ApexCharts 제약으로 경고 마커+툴팁으로 근사.
- `fine` 티어만 실제 표본을 절단하며, `mid/coarse`는 대표 집계 메타를 사용(파일 크기 절감).
- 단일 파일 약 10MB(프로덕션 gzip 시 ~2MB); 더 긴 보존/해상도는 생성기 파라미터 조정 필요.
